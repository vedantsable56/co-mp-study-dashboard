## Q11. Draw the Pin Diagram of 8086 and Explain ALE, WR̅, DEN̅ and DT/R̅ (6 Marks)

### Introduction:
*   The 8086 is a 16-bit third-generation microprocessor designed by Intel.
*   It features a 20-bit address bus, allowing it to access a maximum of 1 MB (2^20 bytes) of physical memory.
*   It has a 16-bit data bus, allowing it to read or write 16 bits of data in a single operation.
*   To reduce the physical pin count of the 40-pin Integrated Circuit (IC), the 8086 uses a multiplexed address and data bus (AD0–AD15) where pins are shared for address and data signals at different times.

---

### Pin Diagram (Draw in Exam):


---

### Functions of Important Pins (Write in Detail):

1.  **ALE (Address Latch Enable)**:
    *   **Description**: This is an active-high output signal generated by the 8086 processor during the first clock state (T1) of every bus cycle.
    *   **Function**: It is used to signal external hardware that a valid physical address is present on the multiplexed AD0–AD15 lines.
    *   **External connection**: ALE is connected directly to the Enable (G) pin of an external latch IC (such as the 74LS373). When ALE pulses HIGH, the latch stores the address. When ALE goes LOW, the address remains locked in the latch, freeing the AD0–AD15 lines to carry data. This is crucial for separating the address and data signals.

2.  **WR̅ (Write)**:
    *   **Description**: This is an active-low output control signal (indicated by the bar over WR).
    *   **Function**: It is driven LOW by the 8086 CPU during the T2 clock state of a write bus cycle, indicating that the CPU has placed valid data on the data bus.
    *   **Operation**: It remains LOW during T3 to allow the memory or I/O device to write the data, and returns HIGH during T4. This transition signals the end of the write operation.

3.  **DEN̅ (Data Enable)**:
    *   **Description**: This is an active-low output control signal.
    *   **Function**: It is driven LOW by the CPU to enable external bidirectional data bus transceivers (such as the 74LS245).
    *   **Operation**: It acts as a gatekeeper to ensure that the CPU data bus is only connected to the external system bus during active read or write phases. This prevents bus conflict by isolating the data lines when the bus is idle or when addresses are being transferred.

4.  **DT/R̅ (Data Transmit/Receive)**:
    *   **Description**: This is an output control signal that determines the direction of data flow through the external bus transceivers.
    *   **Operation**: When DT/R̅ is HIGH, the processor is transmitting data (performing a write operation). When DT/R̅ is LOW, the processor is receiving data (performing a read operation).
    *   **Timing**: The CPU sets the status of DT/R̅ during the T1 phase (before driving DEN̅ LOW) so that the transceivers are configured in the correct direction before data transmission begins.

---

### Other Pins (Briefly Mention in Exam):
*   **BHE̅ (Bus High Enable)**: An active-low output signal. When BHE̅ is 0, it enables the upper memory bank (connected to data lines D8–D15). Combined with address bit A0, it allows the processor to access an 8-bit byte from the even bank, an 8-bit byte from the odd bank, or a full 16-bit word in one cycle.
*   **MN/MX̅ (Minimum/Maximum Mode)**: An input pin that defines the operating mode of the processor. If tied to +5V (Logic 1), the CPU runs in Minimum Mode (single-processor setup, where the 8086 generates all control signals itself). If tied to GND (Logic 0), it runs in Maximum Mode (multiprocessor setup, where control signals are generated by an external 8288 bus controller).

## Q12. Explain Architecture of 8086 (7 Marks)

### Introduction:
*   The architecture of the 8086 microprocessor is split into two independent functional units: the **Bus Interface Unit (BIU)** and the **Execution Unit (EU)**.
*   By separating the execution tasks from the bus operations, the 8086 can perform instruction fetching and execution in parallel. This overlapping of tasks is known as pipelining, which significantly increases the operating speed of the system.

---

### Block Diagram (Draw in Exam):

```
  ┌──────────────────────────────────────────────────────────────────┐
  │                         8086 CPU                                 │
  │                                                                  │
  │  ┌─────────────────────────┐        ┌─────────────────────────┐  │
  │  │   BIU (Bus Interface)   │        │   EU (Execution Unit)   │  │
  │  │  Segment Regs (CS, DS,  │        │  General Regs (AX, BX,  │  │
  │  │  SS, ES) & IP           │        │  CX, DX, SP, BP, SI, DI)│  │
  │  └──────────┬──────────────┘        └────────────▲────────────┘  │
  │             │ 20-bit Address                     │               │
  │             ▼                                    │               │
  │       Address Adder (Σ)                     Control Unit         │
  │             │                                    │               │
  │       ┌─────▼─────┐                         ┌────┴────┐          │
  │       │ 6-Byte    ├─ (Prefetched Opcodes) ─►│   ALU   │          │
  │       │ Queue     │                         └─────────┘          │
  │       └─────┬─────┘                                              │
  └─────────────┼────────────────────────────────────────────────────┘
                ▼ External System Bus
```

---

### 1. Bus Interface Unit (BIU) - Detailed Functions and Components:

The primary job of the BIU is to handle all data and address transfers between the CPU and the external memory or I/O devices.

*   **Core Functions**:
    *   **Instruction Prefetching**: Fetches instruction opcodes from memory before they are needed by the execution unit.
    *   **Bus Management**: Performs memory read/write cycles and I/O port read/write operations.
    *   **Physical Address Generation**: Combines segment bases and offsets to construct 20-bit physical addresses.
*   **Key Components**:
    *   **Segment Registers (CS, DS, SS, ES)**: 16-bit registers that define the base addresses of four active 64 KB memory segments:
        *   *CS (Code Segment)*: Holds program instructions.
        *   *DS (Data Segment)*: Holds static and global variables.
        *   *SS (Stack Segment)*: Holds temporary addresses and subroutine return pointers.
        *   *ES (Extra Segment)*: Used as a destination segment for string operations.
    *   **Instruction Pointer (IP)**: A 16-bit register that holds the offset address of the next instruction to be fetched from the code segment. The BIU automatically updates the IP after each fetch.
    *   **6-Byte Instruction Queue**: A First-In-First-Out (FIFO) buffer that stores up to 6 bytes of instruction codes. This allows the BIU to fetch instructions in advance while the EU is busy executing.
    *   **Address Adder (Σ)**: An internal hardware circuit that performs a single shift-and-add operation to generate 20-bit physical addresses:
        Physical Address = (Segment Address x 10H) + Offset Address

---

### 2. Execution Unit (EU) - Detailed Functions and Components:

The primary job of the EU is to decode and execute the instructions stored in the BIU queue.

*   **Core Functions**:
    *   **Instruction Decoding**: Translates instruction opcodes into internal control signals.
    *   **Execution**: Performs arithmetic, logic, and data transfer operations.
    *   **Flag Management**: Updates the status flags based on the results of operations.
*   **Key Components**:
    *   **ALU (Arithmetic Logic Unit)**: A 16-bit logic unit that performs operations like addition, subtraction, multiplication, division, AND, OR, XOR, and bit shifts.
    *   **General Purpose Registers**: 16-bit registers (AX, BX, CX, DX) used to store operands and temporary calculation data. They can also be accessed as 8-bit registers (AH, AL, BH, BL, etc.).
    *   **Pointer and Index Registers (SP, BP, SI, DI)**: Used to hold offset addresses for stack operations and data indexing.
    *   **Instruction Decoder**: Reads instruction bytes from the BIU queue, decodes them, and coordinates the execution steps.
    *   **Flag Register**: A 16-bit register containing 9 active flags (6 status flags and 3 control flags) that record the results of ALU operations and control CPU behavior.

---

### Pipelining Concept:
*   In older microprocessors, the CPU had to fetch an instruction, decode it, and execute it before fetching the next one.
*   The 8086 bypasses this sequential delay. While the EU is busy executing a fetched instruction, the BIU uses idle bus cycles to prefetch the next instructions into the 6-byte queue.
*   **Queue Flush Exception**: When a branch instruction (like `JMP` or `CALL`) is executed, the prefetched instructions in the queue are no longer valid. The EU flushes (clears) the queue, and the BIU must fetch the instruction from the new target address, which introduces a small delay.

## Q13. Explain Programmer's Model (Register Organization) of 8086 (7 Marks)

### Introduction:
*   The register organization of the 8086 microprocessor is a programmer's model containing 14 user-accessible 16-bit registers.
*   These registers are divided into four main categories: General Purpose Registers, Segment Registers, Pointer & Index Registers, and the Flag Register.

---

### Register Layout Diagram (Draw in Exam):

```
  AX: [  AH  ][  AL  ]   (Accumulator)     CS: [   Code Segment   ]
  BX: [  BH  ][  BL  ]   (Base)            DS: [   Data Segment   ]
  CX: [  CH  ][  CL  ]   (Count)           SS: [   Stack Segment  ]
  DX: [  DH  ][  DL  ]   (Data)            ES: [   Extra Segment  ]

  SP: [ Stack Pointer ]                    IP: [Inst. Pointer     ]
  BP: [ Base Pointer  ]                  
  SI: [ Source Index  ]                    FLAGS: [Status & Control]
  DI: [ Dest. Index   ]
```

---

### 1. General Purpose Registers (AX, BX, CX, DX):
These registers can be used as 16-bit registers or split into two independent 8-bit registers (High byte and Low byte) to handle byte-sized data.

*   **AX (Accumulator Register - AH/AL)**: Used as the primary register for arithmetic and logical operations, input/output port operations, and word multiplication or division.
*   **BX (Base Register - BH/BL)**: Used to hold a base address pointer. It is the only general-purpose register that can be used to hold memory offsets in register indirect addressing modes.
*   **CX (Count Register - CH/CL)**: Used as a loop counter for the `LOOP` instruction, and as a shift/rotate counter for multi-bit shift operations.
*   **DX (Data Register - DH/DL)**: Used to hold the 16-bit I/O port address during port operations, and holds high-order bits of products/dividends during 32-bit arithmetic.

---

### 2. Segment Registers (CS, DS, SS, ES):
These 16-bit registers hold the base addresses of 64 KB memory segments, allowing the 8086 to access different portions of the 1 MB address space.

*   **CS (Code Segment)**: Holds the starting address of the segment containing the program instructions. The IP register provides the offset.
*   **DS (Data Segment)**: Points to the segment where program variables and static data are stored. BX, SI, or DI registers usually provide the offset.
*   **SS (Stack Segment)**: Points to the segment used for temporary storage, subroutine arguments, and return addresses. SP or BP registers provide the offset.
*   **ES (Extra Segment)**: Used by string instructions as a destination data segment. The DI register provides the offset.

---

### 3. Pointer and Index Registers (SP, BP, SI, DI, IP):
These 16-bit registers are used to hold offset addresses for accessing memory.

*   **SP (Stack Pointer)**: Points to the current top of the stack. It is automatically decremented during a `PUSH` operation and incremented during a `POP` operation.
*   **BP (Base Pointer)**: Primarily used to access data parameters passed to subroutines on the stack.
*   **SI (Source Index)**: Holds the offset address of source data in the data segment during string operations.
*   **DI (Destination Index)**: Holds the offset address of destination data in the extra segment during string operations.
*   **IP (Instruction Pointer)**: Holds the offset of the next instruction to be fetched from the code segment. The programmer cannot modify it directly.

---

### 4. Flag Register (Status & Control Flags):
A 16-bit register containing 9 active flags (6 status flags that reflect the result of the last arithmetic/logic operation, and 3 control flags that control CPU operation):

*   **Status Flags**:
    *   *CF (Carry Flag)*: Set to 1 if there is a carry out of the MSB during addition, or a borrow during subtraction.
    *   *PF (Parity Flag)*: Set to 1 if the lower 8 bits of the result contain an even number of 1s (even parity).
    *   *AF (Auxiliary Carry Flag)*: Set to 1 if there is a carry from bit 3 to bit 4 during BCD arithmetic.
    *   *ZF (Zero Flag)*: Set to 1 if the result of an operation is zero.
    *   *SF (Sign Flag)*: Set to 1 if the MSB of the result is 1 (indicating a negative number).
    *   *OF (Overflow Flag)*: Set to 1 if a signed arithmetic operation exceeds the capacity of the register.
*   **Control Flags**:
    *   *TF (Trap Flag)*: Set to 1 to enable single-step debugging mode. The CPU will generate an internal interrupt after executing each instruction.
    *   *IF (Interrupt Flag)*: Set to 1 to enable maskable interrupts via the INTR pin. Set to 0 to disable them.
    *   *DF (Direction Flag)*: Set to 1 to auto-decrement index registers (SI, DI) during string operations, or set to 0 to auto-increment them.

## Q14. Explain Immediate, Register, Direct and Indirect Addressing Modes (7 Marks)

### Definition:
*   An addressing mode is the method used by the instruction to specify where the operand (data) is located.
*   The 8086 provides multiple addressing modes to support variables, arrays, pointers, and stack arguments.

---

### Addressing Modes Comparison Table:

| Comparison Point | Immediate Mode | Register Mode | Direct Mode | Register Indirect Mode |
| :--- | :--- | :--- | :--- | :--- |
| **1. Operand Location** | Inside the instruction code. | Inside a CPU register. | Inside a memory location. | Inside a memory location. |
| **2. Address Specification** | No address is specified. | Register name is specified. | 16-bit offset address in brackets. | Pointer register (BX/SI/DI/BP) in brackets. |
| **3. Memory Access Cycles**| 0 cycles (fastest). | 0 cycles (very fast). | 1 cycle (reads from RAM). | 1 cycle (reads from RAM). |
| **4. Physical Address Math**| None. | None. | PA = (DS x 10H) + Offset | PA = (DS x 10H) + [Register] |
| **5. Execution Speed** | Extremely fast. | Extremely fast. | Moderate. | Moderate. |
| **6. Register Required** | None. | Yes (source and/or dest). | None. | Yes (BX, SI, DI, or BP). |
| **7. Typical Application** | Initializing constants. | High-speed calculations. | Accessing single variables. | Iterating through arrays/loops. |
| **8. Assembly Example** | `MOV AX, 1234H` | `MOV AX, BX` | `MOV AX, [1234H]` | `MOV AX, [SI]` |

---

### 1. Immediate Addressing Mode:
*   **Theory**: The operand data is constant and is stored as part of the machine code. There is no memory access required to fetch the operand, which results in fast execution.
*   **Limitation**: Can only be used for the source operand, never as the destination.
*   **Example**:
    ```assembly
    MOV AX, 5000H  ; Copies the hex value 5000H directly into the AX register
    ```

### 2. Register Addressing Mode:
*   **Theory**: The source and destination operands are stored in CPU registers. This mode is extremely fast because it does not access the external system bus.
*   **Example**:
    ```assembly
    MOV AX, BX     ; Copies the 16-bit content of register BX into register AX
    ```

### 3. Direct Addressing Mode:
*   **Theory**: The 16-bit offset address of the memory operand is written directly in brackets. The CPU combines this offset with the DS register (unless a segment override is used) to calculate the physical memory address.
*   **Example**:
    ```assembly
    MOV AX, [1234H] ; Copies the word from memory offset 1234H into AX
    ```
*   **Physical Address Calculation**:
    If Segment register DS = 1000H, and offset = 1234H:
    1. Shift DS left by 4 bits: 1000H x 10H = 10000H
    2. Add offset: 10000H + 1234H = 11234H
    3. The CPU reads a 16-bit word starting at physical address 11234H.

### 4. Register Indirect Addressing Mode:
*   **Theory**: The offset address of the memory operand is stored in a pointer or index register (BX, BP, SI, or DI). The instruction references the register inside brackets. This is highly useful for loops and array operations.
*   **Example**:
    ```assembly
    MOV AX, [SI]    ; Copies the word from the memory address pointed to by SI into AX
    ```
*   **Physical Address Calculation**:
    If DS = 2000H, and SI = 0050H:
    1. Shift DS left by 4 bits: 2000H x 10H = 20000H
    2. Add SI content: 20000H + 0050H = 20050H
    3. The CPU reads a 16-bit word starting at physical address 20050H.

## Q15. How is 8086 Instruction Set Classified? Explain Any Two Categories (6 Marks)

### Classification of 8086 Instruction Set:
The 8086 has a rich set of instructions that are classified into six major categories based on their function:
1.  **Data Transfer Instructions**: Move data between registers, memory, and I/O ports.
2.  **Arithmetic Instructions**: Perform mathematical operations like addition, subtraction, multiplication, and division.
3.  **Logical Instructions**: Perform operations like AND, OR, XOR, shifts, and rotations.
4.  **String Manipulation Instructions**: Handle operations on blocks of bytes or words (e.g., compare, scan, copy).
5.  **Program Control Instructions**: Control program flow (jumps, calls, loops, and returns).
6.  **Processor Control Instructions**: Control processor states and external bus actions (e.g., lock, halt, set/clear flags).

---

### Detailed Explanation of Any Two Categories (Write in Exam):

#### 1. Data Transfer Instructions:
*   **Purpose**: These instructions are used to copy data from a source location to a destination location. The source can be an immediate value, a register, or a memory location, while the destination can be a register or a memory location.
*   **Flag Status**: An important characteristic of data transfer instructions is that they **do not affect any status flags** in the Flag Register.
*   **Common Instructions**:
    *   `MOV`: Copies byte or word from source to destination.
    *   `PUSH`: Pushes a 16-bit word onto the stack. Decrements the SP register by 2.
    *   `POP`: Pops a 16-bit word from the stack. Increments the SP register by 2.
    *   `IN` / `OUT`: Transfers data between the AL/AX register and an I/O port.
*   **Examples**:
    ```assembly
    MOV AX, BX   ; Copy the 16-bit contents of BX to AX
    PUSH AX      ; Save AX contents onto the stack; SP decreases by 2
    POP DX       ; Retrieve the top stack value and store it in DX; SP increases by 2
    IN AL, 80H   ; Read an 8-bit byte from I/O port 80H into the AL register
    ```

#### 2. Arithmetic Instructions:
*   **Purpose**: These instructions perform basic mathematical operations on binary or BCD data.
*   **Flag Status**: Unlike data transfer instructions, arithmetic operations **update the status flags** (CF, PF, AF, ZF, SF, OF) to reflect the result.
*   **Common Instructions**:
    *   `ADD`: Adds source to destination and stores the result in destination.
    *   `SUB`: Subtracts source from destination and stores the result in destination.
    *   `INC` / `DEC`: Increments or decrements a register or memory location by 1. Note that `INC` and `DEC` do not affect the Carry Flag (CF).
    *   `CMP`: Compares two operands by subtracting the source from the destination. It updates the flags but does not modify either operand.
*   **Examples**:
    ```assembly
    ADD AX, BX   ; AX = AX + BX. Updates all status flags based on the result.
    SUB AX, CX   ; AX = AX - CX.
    INC DX       ; DX = DX + 1. (Updates ZF, SF, OF, PF, AF; CF is not affected).
    CMP AX, 2000H; Subtracts 2000H from AX to set status flags, AX remains unchanged.
    ```

---

### Conclusion:
*   The 8086 instruction set is organized into six classes. Data transfer instructions allow moving data without altering status flags, whereas arithmetic instructions perform operations and modify the status flags to reflect the arithmetic outcome.
