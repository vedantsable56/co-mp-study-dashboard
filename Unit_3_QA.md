## Q11. Draw the Pin Diagram of 8086 and Explain ALE, WR̅, DEN̅ and DT/R̅ (6 Marks)

### Definition
The pin configuration of the 8086 microprocessor is a standard 40-pin Dual In-line Package (DIP) layout. To reduce the package size, the processor uses multiplexed address and data bus lines (AD0–AD15), requiring external hardware support and dedicated control signals to separate the address and data phases during a bus cycle.

### Diagram
```
                     ┌────────────┐
         GND  [1]  ──┤            ├──  [40] VCC (+5V)
     AD14     [2]  ──┤            ├──  [39] AD15
     AD13     [3]  ──┤            ├──  [38] A16/S3
     AD12     [4]  ──┤            ├──  [37] A17/S4
     AD11     [5]  ──┤            ├──  [36] A18/S5
     AD10     [6]  ──┤    8086    ├──  [35] A19/S6
     AD9      [7]  ──┤            ├──  [34] BHE̅/S7
     AD8      [8]  ──┤    CPU     ├──  [33] MN/MX̅
     AD7      [9]  ──┤            ├──  [32] RD̅
     AD6      [10] ──┤            ├──  [31] HOLD
     AD5      [11] ──┤            ├──  [30] HLDA
     AD4      [12] ──┤            ├──  [29] WR̅
     AD3      [13] ──┤            ├──  [28] M/IO̅
     AD2      [14] ──┤            ├──  [27] DT/R̅
     AD1      [15] ──┤            ├──  [26] DEN̅
     AD0      [16] ──┤            ├──  [25] ALE
     NMI      [17] ──┤            ├──  [24] INTA̅
     INTR     [18] ──┤            ├──  [23] TEST̅
     CLK      [19] ──┤            ├──  [22] READY
     GND      [20] ──┤            └──────  [21] RESET
                     └────────────┘
```

### Detailed Explanation of Pin Functions
*   **ALE (Address Latch Enable)**: This is an active-high output signal generated by the CPU during the first clock state (T1) of every bus cycle. It signals external latch devices (such as the 74LS373 IC) to lock the valid address bits present on the multiplexed AD0–AD15 lines. When ALE drops low, the address is held in the latch, freeing the CPU lines to carry data.
*   **WR̅ (Write)**: This is an active-low output control signal driven low by the processor during the T2 clock phase of a write bus cycle. It indicates that the CPU has placed valid data on the data bus lines, and it remains active through T3 to allow memory or I/O devices to write the data.
*   **DEN̅ (Data Enable)**: This is an active-low output control signal used to enable external bidirectional data transceivers (such as the 74LS245 IC). It goes low during the T2 phase of read and write cycles to connect the CPU's internal data bus to the external system bus, preventing bus conflicts when address signals are active.
*   **DT/R̅ (Data Transmit/Receive)**: This is an output control signal that defines the direction of data transmission through the external transceivers. When DT/R̅ is driven high, the transceivers transmit data from the CPU to the bus (write cycle); when driven low, they receive data from the bus to the CPU (read cycle).
*   **BHE̅ (Bus High Enable)**: This is an active-low output signal used to enable the upper memory bank (connected to data lines D8–D15). Combined with address bit A0, it allows the processor to perform 8-bit transfers on the upper data lines, 8-bit transfers on the lower data lines, or 16-bit word transfers.
*   **MN/MX̅ (Minimum/Maximum Mode)**: This is an input pin that configures the operating mode of the 8086 processor. When tied to VCC (+5V), the CPU operates in Minimum Mode (single-processor setup), generating its own control bus signals. When connected to GND, it operates in Maximum Mode (multiprocessor setup), relying on the external 8288 bus controller to generate control signals.

### Key Features
*   **Multiplexed Bus Architecture**: Combines address and data signals on the same physical lines to reduce the pin count of the processor package.
*   **Tristate Capability**: Many pins support a high-impedance state, allowing external devices like DMA controllers to take control of the system bus.

## Q12. Explain Architecture of 8086 (7 Marks)

### Definition
The architecture of the 8086 microprocessor is split into two independent functional blocks: the **Bus Interface Unit (BIU)** and the **Execution Unit (EU)**. This modular division enables parallel execution of instructions (pipelining), as the BIU fetches instructions in advance while the EU decodes and executes the currently loaded instruction.

### Labelled Diagram
```
  ┌──────────────────────────────────────────────────────────────────┐
  │                         8086 CPU                                 │
  │                                                                  │
  │  ┌─────────────────────────┐        ┌─────────────────────────┐  │
  │  │   Bus Interface Unit    │        │     Execution Unit      │  │
  │  │  Segment Regs (CS, DS,  │        │  General Regs (AX, BX,  │  │
  │  │  SS, ES) & IP           │        │  CX, DX, SP, BP, SI, DI)│  │
  │  └──────────┬──────────────┘        └────────────▲────────────┘  │
  │             │ 20-bit Address                     │               │
  │             ▼                                    │               │
  │       Address Adder (Σ)                     Control Unit         │
  │             │                                    │               │
  │       ┌─────▼─────┐                         ┌────┴────┐          │
  │       │ 6-Byte    ├─ (Prefetched Opcodes) ─►│   ALU   │          │
  │       │ Queue     │                         └─────────┘          │
  │       └─────┬─────┘                                              │
  └─────────────┼────────────────────────────────────────────────────┘
                ▼ External System Bus
```

### Explanation of Architectural Blocks
*   Bus Interface Unit (BIU):
    *   *Segment Registers (CS, DS, SS, ES)*: These 16-bit registers hold the base addresses of four active memory segments (Code, Data, Stack, Extra), allowing the CPU to partition memory blocks.
    *   *Instruction Pointer (IP)*: A 16-bit register holding the offset address of the next instruction byte to be fetched from the code segment.
    *   *Address Generation Adder (Σ)*: An internal hardware adder that shifts the 16-bit segment address left by 4 bits and adds the 16-bit offset to produce a 20-bit physical address.
    *   *6-Byte Instruction Queue*: A First-In-First-Out (FIFO) buffer that stores up to 6 bytes of prefetched instruction code, enabling parallel instruction fetching.
*   Execution Unit (EU):
    *   *Arithmetic Logic Unit (ALU)*: A 16-bit arithmetic and logic block that executes mathematical operations (ADD, SUB, MUL, DIV) and Boolean logical operations (AND, OR, XOR).
    *   *General Purpose Registers (AX, BX, CX, DX)*: Registers used for data manipulation. They can be split into independent 8-bit registers (AH, AL, etc.) for byte operations.
    *   *Pointer and Index Registers (SP, BP, SI, DI)*: Store offset addresses for stack access (SP, BP) and string manipulation indexing (SI, DI).
    *   *Instruction Decoder & Control Unit*: Reads instruction bytes from the BIU queue, decodes the opcodes, and generates internal control signals to execute the operations.
    *   *Flag Register*: A 16-bit register containing 9 active flags (6 status flags and 3 control flags) that record the results of ALU operations and define CPU operating modes.

### Working of Architecture
The BIU and EU operate in parallel. When the system is initialized, the BIU calculates physical addresses, fetches instruction bytes from memory, and stores them in the 6-byte queue. The EU continuously pops decoded instructions from the queue and executes them. If the ALU requires memory access for operands, the EU requests the BIU to perform a read or write bus cycle. When a branch instruction (e.g., JMP or CALL) is executed, the EU flushes the instruction queue, and the BIU immediately fetches the instruction from the new target address, introducing a queue-refilling delay.

### Key Features
*   **Parallel Pipelining**: Prefetches up to 6 bytes of instruction codes during idle bus cycles, reducing instruction execution delays.
*   **Segmented Memory Support**: Divides the 1 MB physical address space into logical segments, allowing relocatable code and data isolation.

## Q13. Explain Programmer's Model (Register Organization) of 8086 (7 Marks)

### Definition
The programmer's model of the 8086 microprocessor consists of 14 user-accessible 16-bit registers. These registers are organized into distinct functional categories to manage data processing, memory segmentation, stack operations, and processor status flag configurations.

### Labelled Diagram
```
  AX: [  AH  ][  AL  ]   (Accumulator)     CS: [   Code Segment   ]
  BX: [  BH  ][  BL  ]   (Base)            DS: [   Data Segment   ]
  CX: [  CH  ][  CL  ]   (Count)           SS: [   Stack Segment  ]
  DX: [  DH  ][  DL  ]   (Data)            ES: [   Extra Segment  ]

  SP: [ Stack Pointer ]                    IP: [Inst. Pointer     ]
  BP: [ Base Pointer  ]                  
  SI: [ Source Index  ]                    FLAGS: [Status & Control]
  DI: [ Dest. Index   ]
```

### Explanation of Registers
*   General Purpose Registers:
    *   *AX (Accumulator Register - AH/AL)*: Used in word and byte arithmetic operations, input/output port transfers, and multiplication/division calculations.
    *   *BX (Base Register - BH/BL)*: Holds base offset addresses for indexing memory variables. It is the only general-purpose register that can be used for indirect memory addressing.
    *   *CX (Count Register - CH/CL)*: Acts as an automatic loop counter for the `LOOP` instruction and a shift/rotate counter for multi-bit shifts.
    *   *DX (Data Register - DH/DL)*: Holds I/O port addresses during port instructions and stores high-order results during word multiplication/division.
*   Segment Registers:
    *   *CS (Code Segment)*: Holds the 16-bit base address of the code segment containing program instructions.
    *   *DS (Data Segment)*: Points to the base address of the data segment where static variables and program data are stored.
    *   *SS (Stack Segment)*: Holds the base address of the stack segment used for return addresses and temporary parameter storage.
    *   *ES (Extra Segment)*: Points to the extra segment, primarily used as a destination segment for string operations.
*   Pointer and Index Registers:
    *   *SP (Stack Pointer)*: Stores the 16-bit offset of the top of the stack, decrementing on PUSH and incrementing on POP operations.
    *   *BP (Base Pointer)*: Used to access local variables and parameters passed onto the stack during subroutine calls.
    *   *SI (Source Index)*: Stores the source offset address in the data segment for string operations and index addressing.
    *   *DI (Destination Index)*: Stores the destination offset address in the extra segment for string operations.
*   Flag Register (Status and Control Flags):
    *   *Status Flags (CF, PF, AF, ZF, SF, OF)*: Reflect the arithmetic or logical outcome of the last ALU operation (e.g., carry, zero result, overflow, sign status).
    *   *Control Flags (TF, IF, DF)*: Define operating modes, including single-step trap debugging (TF), enabling maskable interrupts (IF), and string increment direction (DF).

### Key Features
*   **8-bit and 16-bit Access**: General-purpose registers can be accessed as single 16-bit registers or split into two independent 8-bit registers (High/Low) to handle byte data.
*   **Segment Offset Cooperation**: Segments combine with dedicated offset registers (CS:IP, DS:SI, SS:SP, ES:DI) to generate 20-bit physical memory addresses.

## Q14. Explain Immediate, Register, Direct and Indirect Addressing Modes (7 Marks)

### Definition / Introduction
Addressing modes define the operational logic used by the CPU to locate and access operands (data) for instruction execution. The 8086 provides different addressing modes to support access to constants, registers, variables, and data structures like arrays or stack frames.

### Diagram
```
  Immediate: [ Opcode ] [ Direct Operand Data ]
  Register:  [ Opcode ] [ Reg Source ] ──► [ Reg Dest ]
  Direct:    [ Opcode ] [ 16-Bit Memory Offset ] ──► RAM Address
  Indirect:  [ Opcode ] [ Reg Pointer (BX/SI/DI) ] ──► Address in Reg ──► RAM Address
```

### Detailed Explanation of Addressing Modes
*   Immediate Addressing Mode:
    *   *Logic*: The operand is a constant value included directly within the instruction code. The CPU does not perform any memory cycles to fetch the data.
    *   *Application*: Used to initialize registers with constant values or load constants for math operations.
    *   *Example*: `MOV AX, 1234H` (Loads the 16-bit hex constant 1234H directly into Register AX).
*   Register Addressing Mode:
    *   *Logic*: The operand data is stored in a CPU register, and the instruction specifies the source and destination registers. It executes entirely within the CPU, requiring zero memory access cycles.
    *   *Application*: Used for high-speed arithmetic and logic operations where operands are already loaded into registers.
    *   *Example*: `MOV AX, BX` (Copies the 16-bit contents of register BX into register AX).
*   Direct Addressing Mode:
    *   *Logic*: The 16-bit offset address of the memory operand is written directly in the instruction. The CPU shifts the Data Segment (DS) register left by 4 bits and adds this displacement to find the physical address.
    *   *Application*: Used to access static variables that have fixed locations in memory.
    *   *Example*: `MOV AX, [5000H]` (Reads a word from the physical address calculated as (DS x 10H) + 5000H and stores it in AX).
*   Register Indirect Addressing Mode:
    *   *Logic*: The offset address of the memory operand is stored in a pointer or index register (BX, BP, SI, or DI). The instruction references the register in brackets, and the CPU reads the address from that register.
    *   *Application*: Used to iterate through arrays, loops, and data blocks where memory offsets change dynamically during execution.
    *   *Example*: `MOV AX, [SI]` (Reads a word from the physical address calculated as (DS x 10H) + SI and stores it in AX).

### Key Features
*   **Flexible Access**: Allows programs to access constants, registers, static memory locations, or dynamic arrays.
*   **Effective Address Generation**: Combines segment bases with index values, base values, and displacements to support complex data structures.

## Q15. How is 8086 Instruction Set Classified? Explain Any Two Categories (6 Marks)

### Definition / Introduction
The 8086 instruction set consists of the machine commands that the CPU can execute. These instructions are classified into six categories based on their function, allowing programmers to perform data movement, mathematical calculations, logical processing, program branching, and system state control.

### Diagram
```
                          ┌──────────────────────────┐
                          │   8086 Instruction Set   │
                          └────────────┬─────────────┘
          ┌──────────────┬─────────────┼──────────────┬──────────────┐
          ▼              ▼             ▼              ▼              ▼
     ┌────────┐     ┌────────┐    ┌────────┐     ┌────────┐     ┌────────┐
     │Data    │     │Arith-  │    │Logical │     │Program │     │Proces- │
     │Transfer│     │metic   │    │& Shift │     │Control │     │sor Ctrl│
     └────────┘     └────────┘    └────────┘     └────────┘     └────────┘
```

### Detailed Explanation of Categories
*   Data Transfer Category:
    *   *Purpose*: Used to copy bytes or words of data between registers, memory locations, and I/O ports. These instructions do not modify the data value.
    *   *Flag Impact*: None of the status flags in the flag register are affected by data transfer operations.
    *   *Instructions*: Includes `MOV` (move data), `PUSH`/`POP` (stack access), `IN`/`OUT` (I/O port access), and `XCHG` (exchange register values).
    *   *Examples*:
        *   `MOV AX, [SI]` (Copies a 16-bit word from the memory location pointed to by SI into Register AX).
        *   `PUSH BX` (Decrements the stack pointer SP by 2 and writes the contents of BX onto the stack).
*   Arithmetic Category:
    *   *Purpose*: Performs binary, packed BCD, or unpacked BCD mathematical calculations on registers or memory data.
    *   *Flag Impact*: Updates the status flags (CF, PF, AF, ZF, SF, OF) to record the result of the operation.
    *   *Instructions*: Includes `ADD`/`SUB` (addition/subtraction), `MUL`/`DIV` (multiplication/division), `INC`/`DEC` (increment/decrement), and `CMP` (compare by subtraction).
    *   *Examples*:
        *   `ADD AX, BX` (Adds the contents of BX to AX, writes the sum to AX, and updates status flags).
        *   `CMP AL, 05H` (Subtracts 05H from AL to update status flags, leaving AL unmodified).
*   Other Categories (Logical, Branch, and Control):
    *   *Logical*: Performs Boolean logic operations (AND, OR, XOR, NOT) and shifts/rotates bit values.
    *   *Program Control*: Manages program branching using conditional/unconditional jumps, subroutine calls, and loops.
    *   *Processor Control*: Modifies CPU flag states (e.g., CLC to clear carry) and coordinates bus operations (e.g., LOCK, HLT).

### Key Features
*   **Flag Management**: Data transfer instructions preserve the flag register, while arithmetic and logical operations update flag status to allow conditional branching.
*   **Variable Data Sizes**: Most instructions support operations on both 8-bit bytes and 16-bit words.
